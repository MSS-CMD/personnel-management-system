# 人员管理平台 — 起步执行清单（单人开发 · 轻量自研单服务）

> 配套文件：`pms-lite/`（已搭好的可运行项目）、计划 `toasty-aurora-newton.md`（轻量方案）
> 角色：你一人开发。旧方案 RuoYi 计划 `quantum-aurora-darwin.md` 保留备用。
> 技术栈：FastAPI + SQLite + 极简前端。**你只需装 Python 3.10+**，其余零安装。

---

## Day 0 — 装环境（一次性，约 5–10 min）
- [ ] 装 **Python 3.10+**（建议 3.11）：官网下载或系统包管理器，SQLite 已内置无需另装。
- [ ] 装依赖（一条命令，依赖已压到最轻，不含 pandas）：
  ```bash
  cd pms-lite
  pip install -r requirements.txt
  ```
- [ ] 确认版本：`python --version` 显示 3.10+ 即 OK。

## Day 1 — 跑起来（Epic0）
- [ ] 启动服务（首次运行自动建表 + 写入 PRD 的 A/B/C 演示数据）：
  ```bash
  python run.py          # 或：uvicorn app.main:app --port 8000
  ```
- [ ] 浏览器开 `http://localhost:8000`，用 `admin / admin123` 登录，看到"核算看板"即底座 OK。
- [ ] 跑本地核算验证，核对 PRD 例子：
  ```bash
  python verify.py      # 预期：A=5000 / A历史=2400 / B=2400 / C=600 / 项目2 超支，全部 ✅
  ```

## Day 2 — 主数据（Epic1：部门/员工/项目/日单价历史）
- [ ] 前端"员工/项目/日单价"页可增删查；员工即登录账号（角色：admin/project_leader/dept_leader/employee）。
- [ ] **单价修改仅 admin**：前端非 admin 隐藏"新增/修改"按钮；后端 `POST /unit-prices` 仅 admin 可调，非 admin 返回 403（已验证 AC-15）。
- [ ] 同一员工同一生效日单价唯一（`uk_emp_eff` 唯一索引，重复报错）。

## Day 3 — 核算核心 + 历史追溯（Epic2）
- [ ] 服务记录保存时后端算 `days = 截止−起始+1`，并按**服务起始日**匹配当时生效单价（取 `effective_date ≤ 起始日` 最新一条）。
- [ ] 无匹配单价 → `price_missing=true`，产值记 0 并在看板置顶（AC-13 历史追溯已验证：A 的 3 月服务按 400 而非 7 月的 500 核算）。
- [ ] 个人产值 = 匹配单价 × days；项目人工 = Σ 该项目产值。

## Day 4 — 成本明细 + 预算管控（Epic3）
- [ ] "成本明细"页录入项目其他成本（差旅/设备/外包等）。
- [ ] 核算看板自动合计 `项目总支出 = 人工 + 其他成本`，超支项目标红（演示中 项目2 因设备租赁 3000 超预算 5000 → 超支）。

## Day 5–D6 — 四角色权限 + 极简前端（Epic4–Epic6）
- [ ] 权限：admin 管全部主数据/单价/预算/导入；项目负责人管自己项目的服务记录与成本明细；普通员工/部门负责人只读。
- [ ] 前端单文件 `app/static/index.html`（原生 JS，响应式，移动端可用），无需打包。

## Day 7 — 导入工具（Epic6 导入）
- [ ] "导入"页支持 **CSV（标准库）/ XLSX（openpyxl）** 批量导入员工、项目、日单价、服务记录、成本明细。
- [ ] 表头字段见页面提示；导入后服务记录自动触发核算。
