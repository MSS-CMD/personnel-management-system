# 本地启动与试玩指南（给非专业开发同学）

本指南面向**第一次在自己电脑上跑这个系统**的朋友，照着一步一步做即可，不需要懂编程。

---

## 一、你需要先装一样东西：Python

系统是用 Python 写的，电脑上需要有 **Python 3.10 或更高版本**。

- **确认是否已装**：打开"命令提示符"（Windows）或"终端"（Mac），输入
  ```
  python --version
  ```
  如果显示 `Python 3.10.x` 或更高（如 3.11、3.12），就 OK。
- **没装或版本太低**：去官网 <https://www.python.org/downloads/> 下载安装。
  - Windows 安装时**务必勾选 "Add Python to PATH"**（添加到环境变量），否则后面命令找不到 python。
  - 装完重新打开一次命令行再试 `python --version`。

> 占用空间很小（几十 MB），之前你担心的磁盘空间完全没问题。

---

## 二、拿到代码

三种方式任选其一。推荐**方式 A（git 拉取）**，能随时拿到最新版。

**方式 A：用 git 拉取到本地（推荐）**
1. 本机装好 Git（<https://git-scm.com/downloads>；Windows 安装时一路下一步即可）。
2. 打开命令行（Windows 按 `Win+R` → 输入 `cmd`），执行：
   ```
   git clone https://github.com/MSS-CMD/personnel-management-system.git
   ```
   会在当前目录生成 `personnel-management-system` 文件夹。
3. 进入程序目录：
   ```
   cd personnel-management-system/pms-lite
   ```
   > 程序在仓库的 `pms-lite/` 子文件夹里，不要进错到根目录。

**方式 B：从 GitHub 下载 ZIP（不用装 Git）**
1. 浏览器打开仓库：<https://github.com/MSS-CMD/personnel-management-system>
2. 点绿色的 **Code → Download ZIP**，解压到任意文件夹（比如 `桌面/pms`）。
3. 进入解压后的 `personnel-management-system/pms-lite` 文件夹。

**方式 C：直接复制本机已有的 `pms-lite` 文件夹**
把 `pms-lite` 整个文件夹拷到你想放的位置即可。

---

## 三、安装依赖（只需做一次）

1. 在命令行里，进入 `pms-lite` 目录：
   ```
   cd 你放的位置/pms-lite
   ```
   > 小技巧：Windows 可以在文件夹地址栏输入 `cmd` 回车，自动就在该目录打开命令行。

2. 安装所需组件（**只在第一次运行时需要**）：
   ```
   pip install -r requirements.txt
   ```
   看到一堆 `Successfully installed ...` 就成功了。如果提示 `pip` 找不到，用 `python -m pip install -r requirements.txt`。

---

## 四、启动系统

在 `pms-lite` 目录下，输入：
```
python run.py
```
看到类似下面这行就说明启动成功：
```
Uvicorn running on http://0.0.0.0:8000
```
> 启动后**这个窗口不要关**，它就是正在运行的服务器。要停止就按 `Ctrl + C`。

---

## 五、打开使用界面

用浏览器（Chrome / Edge 都行）访问：
```
http://127.0.0.1:8000/
```
看到登录页就成功了 🎉

## 六、用演示账号登录试玩

系统第一次启动会自动写入一套演示数据，直接用下面的账号登录：

| 角色 | 用户名 | 密码 | 能做什么 |
|---|---|---|---|
| 系统管理员 | `admin` | `admin123` | **全部功能**，包括改日单价、改预算 |
| 员工 A | `employee_a` | `admin123` | 看自己产值、录服务记录 |
| 员工 B | `employee_b` | `admin123` | 同上 |
| 员工 C | `employee_c` | `admin123` | 同上 |
| 部门负责人 | `dept_head` | `admin123` | 看部门数据 |

> **建议先用 `admin` 登录**，这样能看到全部菜单和看板。

## 七、5 分钟试玩路线（照着点一遍就懂了）

登录 `admin` 后，左侧/顶部有这些菜单：

1. **核算看板**：一眼看到每个项目的人工、其他成本、总支出、是否超支（超支标红）。
   - 预期：项目1 总支出 8800（不超支），项目2 总支出 5400（**超支**，因为设备租赁 3000 超了 5000 预算）。
2. **部门 / 员工**：看看演示里有哪些部门、哪些人。
3. **日单价**：这里记录每人每个生效日期的日单价，且**只有 admin 能改**（这就是你说的"仅管理员可改单价"）。
   - 试试：用 `employee_a` 登录，去日单价页面会看不到"新增/编辑"按钮（权限隔离）。
4. **服务记录**：谁在哪个项目、哪天到哪天在现场干活。系统按"服务天数 = 截止−起始+1"算。
5. **成本明细**：项目除人工外的其他花销（差旅、设备租赁等）。
6. **导入**：可以把 Excel/CSV 批量导入员工、日单价、服务记录、成本明细（大批量录数据用）。

## 八、常见问题

| 现象 | 原因 / 解决 |
|---|---|
| 浏览器打不开 `127.0.0.1:8000` | 命令行窗口关了或没启动成功。回到第四步重跑 `python run.py`，确认看到 `Uvicorn running` 那行。 |
| 提示端口被占用 | 8000 被别的程序占了。改 `run.py` 里的 `port=8000` 为 `port=8080`，访问地址也相应改成 `http://127.0.0.1:8080/`。 |
| 想清空演示数据重来 | 关掉服务，删除 `pms-lite/data/pms.db` 这个文件，再启动就会重新写入演示数据。 |
| 想换成自己的真实数据 | 用 `admin` 登录，在各部门菜单里逐条增删改；大批量用"导入"功能。 |
| 忘记密码 | 目前演示阶段统一是 `admin123`；正式用前请在 `app/security.py` 之外自行管理，或告诉我帮你加"修改密码"功能。 |

## 九、安全提醒（重要）

- 演示账号密码是公开的，**仅用于本地试用**，切勿直接用于正式环境。
- 项目早期用过的 GitHub token 已在对话中暴露，**请到 GitHub 后台把它撤销（revoke）**，避免被他人利用。
- 正式上线前需要：改默认密码、用 HTTPS、把数据库放到安全位置。这些我可以在后续阶段帮你做。
