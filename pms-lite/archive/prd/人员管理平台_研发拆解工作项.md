## 1. Epic 0 — 技术底座与环境（前置，无优先级）

### T0.1 拉取基础框架与本地运行环境
- **主要交付**
  1. 获取 `oa-flow-pms` 仓库（或备选 RuoYi-Vue-Plus），搭建 Java 17 + Spring Boot + Vue3 本地开发环境。
  2. 跑通若依框架默认启动（后端 + 前端 + 数据库），确认本地可访问。
- **验收要点**：本地 `npm run dev` / Maven 构建后，框架默认页面可打开；环境文档（README）记录启动命令。
- **依赖**：无
- **建议角色/指派**：研发负责人 / 后端开发（脚手架）
- **关联 PRD**：NFR-DEPLOY（本地验证部署）

### T0.2 数据库 / 权限框架 / 组织用户初始化
- **主要交付**
  1. 初始化业务库表空间，接入若依权限（菜单/角色/用户/数据权限）体系。
  2. 确认用户-角色-部门基础数据可用，API 层以 REST 暴露（移动端预留接口，本版不实现移动端）。
- **验收要点**：可新建用户并赋予角色；API 可被 Postman/前端调用返回 JSON。
- **依赖**：T0.1
- **建议角色/指派**：后端开发
- **关联 PRD**：FR-ORG、FR-AUTH、NFR-MOBILE（预留接口）

## 2. Epic 1 — 组织与主数据（P0）

### T1.1 部门管理
- **主要交付**：部门 CRUD + 列表/树形展示；支持"公司级/未分配"根节点。
- **验收要点**：AC-1（可建多部门，员工/项目可归属部门）。
- **依赖**：T0.2
- **建议角色/指派**：后端+前端
- **关联 PRD**：FR-ORG、AC-1

### T1.2 员工管理与日单价（含生效日期历史）
- **主要交付**
  1. 员工档案（姓名、部门、岗位、状态[在用/停用]）。
  2. 日单价历史：多条带生效日期的记录；新增/修改写入新生效历史。
  3. **单价修改入口仅对系统管理员开放**（前端按角色隐藏，后端按角色拦截）。
  4. 同生效日期重复单价拒绝（提示"该生效日期已存在单价"）。
  5. 停用后禁止关联新服务记录（历史保留）。
- **验收要点**：AC-2（日单价历史增改、重复生效日拒）、AC-13（历史追溯基础）、B-2/B-3/B-6。
- **依赖**：T0.2、T1.1
- **建议角色/指派**：后端+前端
- **关联 PRD**：FR-EMP、AC-2、AC-13、B-2/B-3/B-6、FR-AUTH(单价仅管理员)

### T1.3 项目管理（归属部门 + 负责人 + 预算）
- **主要交付**：项目档案（名称、归属部门、负责人、预算[默认0]、状态[进行中/已结项]）；创建置"进行中"、执行率 0%；结项冻结录入。
- **验收要点**：AC-3（建项目设预算、结项冻结）。
- **依赖**：T0.2、T1.1
- **建议角色/指派**：后端+前端
- **关联 PRD**：FR-PRJ、AC-3、B-4

## 3. Epic 2 — 服务记录与核算引擎（P0 + 历史追溯 P1）

### T2.1 服务记录录入
- **主要交付**：员工 + 项目 + 起止日期；自动算天数 = 截止−起始+1（含首尾）；校验起始晚于截止、同员工同项目日期重叠。
- **验收要点**：AC-4（A/B/C 天数 10/6/3）。
- **依赖**：T1.2、T1.3
- **建议角色/指派**：后端+前端
- **关联 PRD**：FR-SVC、AC-4、B-1（跨月连续）

### T2.2 核算引擎（核心）
- **主要交付**
  1. 个人人工产值 = 服务日期所匹配生效日单价 × 天数（见 T2.3 追溯匹配）。
  2. 项目人工支出 = Σ 该项目服务记录人工产值；项目其他成本 = Σ 成本明细；项目总支出 = 人工 + 其他；执行率 = 总支出 ÷ 预算。
  3. 任一变更（单价/服务/成本/预算）实时重算受影响值。
  4. 多员工同项目分别累计后求和，与逐条一致。
- **验收要点**：AC-5（A=5000/B=2400/C=600）、AC-7（项目1=6400/项目2=2400）。
- **依赖**：T1.2、T1.3、T2.1
- **建议角色/指派**：后端（核心算法）
- **关联 PRD**：FR-CAL、AC-5、AC-7、B-2/B-4

### T2.3 历史单价追溯
- **主要交付**：核算时按"每条服务记录的日期"匹配当时生效单价（取生效日期 ≤ 服务日期的最新一条）；无匹配标"单价缺失"并置顶提示，不计入直至补全。
- **验收要点**：AC-13（A 3月400/7月500，7月服务按500、历史3月服务按400 分别核算）、B-2、B-6。
- **依赖**：T2.1、T2.2
- **建议角色/指派**：后端
- **关联 PRD**：FR-CAL、AC-13、B-2/B-6

## 4. Epic 3 — 项目成本明细（P1）

### T3.1 成本明细录入
- **主要交付**：项目 + 成本类型[差旅/材料/外包/其他] + 金额 + 发生日期 + 备注；校验金额 ≤0 或类型空拒绝；可选关联员工。
- **验收要点**：AC-6（项目1 录差旅800→其他成本800）、AC-14（项目负责人可录/改所负责项目成本）、B-7。
- **依赖**：T1.3
- **建议角色/指派**：后端+前端
- **关联 PRD**：FR-COST、AC-6、AC-14、B-7

### T3.2 成本聚合
- **主要交付**：其他成本 = Σ 明细；总支出 = 人工 + 其他；重算执行率；成本明细按发生日期记录（支持跨月 B-1）。
- **验收要点**：AC-7（项目1 总支出 = 人工5600+800=6400）。
- **依赖**：T3.1、T2.2
- **建议角色/指派**：后端
- **关联 PRD**：FR-CAL、AC-7

## 7. Epic 6 — 权限与角色（P1）

### T6.1 四角色数据范围隔离
- **主要交付**：统一数据权限切面，按"角色 + 部门 + 项目负责人"三维隔离。
  - 系统管理员：全量。
  - 部门负责人：仅管理部门内员工/项目/服务/成本/产值。
  - 普通员工：仅本人服务记录与人工产值，隐藏他人与预算明细。
  - 项目负责人：仅其负责项目及服务这些项目的员工；**不可修改员工日单价、预算、他人项目**（单价仅管理员）。
- **验收要点**：AC-11（部门/员工隔离）、AC-15（越权拦截 + 无改单价权限）。
- **依赖**：T1.1、T1.3
- **建议角色/指派**：后端（切面）
- **关联 PRD**：FR-AUTH、AC-11、AC-15

### T6.2 项目负责人权限落地
- **主要交付**：项目负责人对其负责项目可录/改成本明细、可查看；前端隐藏其改单价/预算入口，后端二次拦截。
- **验收要点**：AC-14（成本录入）、AC-15（越权 + 无改单价）。
- **依赖**：T6.1、T3.1
- **建议角色/指派**：后端+前端
- **关联 PRD**：FR-AUTH、AC-14、AC-15

## 11. 关键依赖与风险（供研发负责人关注）

- **D1（硬依赖）**：T2.2 核算引擎依赖 T1.2 单价历史 + T2.1 服务记录，第2批优先。

- **R2 单价模型**：已锁定"全局单价 + 管理员修改 + 历史追溯"，无需按项目独立定价。
- **R3 权限粒度**：四角色 + 部门 + 项目负责人三维隔离，T6.1 设计统一数据权限切面（前端隐藏 + 后端拦截双重保险）。
