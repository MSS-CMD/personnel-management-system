## 一、GitHub 同类项目调研结论

按"可二开 + 贴合日单价×天数模型 + 国内工程场景"三个维度筛查，结果如下：

| 项目 | 技术栈 | License | 契合点 | 主要不足 | 是否推荐二开 |
|------|--------|---------|--------|----------|--------------|
| **huangxing2010/oa-flow-pms**（工程项目管理系统） | Java + Vue（基于 RuoYi-Flowable-Plus） | 继承自若依（MIT） | **最贴合**：专为工程公司做项目进度 + 成本控制 + OA，自带项目/任务/合同/财务概念 | 需二次裁剪，Flowable 工作流偏重 | ✅ **首选底座** |
| **RuoYi-Vue-Plus** | Java + Spring Boot + Vue | MIT | 国内最主流快速开发底座，文档多、易加模块，权限/组织/字典现成 | 仅是脚手架，成本模型需自己写 | ✅ 干净底座备选 |
| **Kimai** | PHP + Symfony 6 + MySQL | GPL-3.0 | 工时+费率（用户/项目级费率）+ 发票核心免费，API/插件齐全，Docker 友好 | 费率是"小时制"，要改造成"日单价"；PHP 栈国内团队可能不熟 | ⚠️ 费率模型可借鉴 |
| **ERPNext** | Python + Frappe + Vue | GPL-3.0 | 自带 Project / Timesheet / 项目预算 / 员工成本模块 | 重量级全功能 ERP，GPL 强 copyleft，定制成本高、易过度设计 | ❌ 过度臃肿 |
| **Dolibarr** | PHP | GPL-3.0 | 轻量 ERP/CRM，含工时+报销+项目模块 | 国内生态弱，日单价模型仍需改 | ⚠️ 备选 |
| **valasek/timesheet** | Go + Gin + Vue/Quasar | **禁止商用** | 轻量自托管、项目+顾问报工 | **License 明确禁止商业使用** | ❌ 直接排除 |
| SAP FSM / Staffjoy Suite | 各异 | 各异 | 偏排班调度 | 不解决成本核算 | ❌ 不匹配 |

**调研结论与推荐**
- 最契合你场景的是 **`oa-flow-pms`**（若依系的工程项目管理系统），它已经把"工程公司、进度、成本、OA"这几个概念搭好了，拿回来裁剪最快。
- 若团队想从更干净的基础起步，用 **RuoYi-Vue-Plus** 做底座、自己写"日单价×天数"成本模块，可控性最高。
- Kimai 的"费率+工时+发票"思路值得借鉴，但它是小时制、PHP 栈，改造成日单价模型成本不低。
- **不建议**直接上 ERPNext（太重、GPL 传染风险）；**排除** valasek/timesheet（License 禁商用）。

## 二、需求理解

### 2.1 目标用户与角色

| 角色 | 是谁 | 核心诉求 |
|------|------|----------|
| **管理者（你）** | 电气工程管理岗，常出差 | 实时掌握每位同事在哪些项目、产生了多少成本，合理调度人力 |
| **出差员工** | 和你一样的工程师 | 方便填报自己的行程/服务天数，不用回公司补 |
| **（可选）财务/内勤** | 核算与预算管控 | 审核项目支出、管理预算、导出报表 |

### 2.2 用户痛点（你描述的）

- 大家各自去不同项目现场，公司**不清楚员工当前行程**，无法合理安排项目人力。
- 缺乏统一台账，员工产值、项目支出**靠人工算、易错、滞后**。
- 一个项目有多人服务时，成本分摊**没有系统化核算**。

### 2.3 核心使用场景

1. **录入**：管理员/员工录入"员工日单价 + 行程（起止日期）+ 所属项目"。
2. **核算**：系统按 `日单价 × 服务天数` 自动算个人产值；按项目聚合算项目总支出。
3. **查看**：一张看板展示"员工→在服务项目 / 个人产值 / 项目预算 / 预算执行率"。
4. **管控**：项目预算可设、可超支预警、可调整。

### 2.4 业务目标（可量化）

- **目标 1**：员工行程与项目归属的**录入到可见延迟 ≤ 1 天**（出差现场填报）。
- **目标 2**：产值/项目成本**100% 由系统自动核算**，人工核对时间从"天级"降到"分钟级"。
- **目标 3**：项目预算执行率**实时可见**，超支能预警。

### 2.5 核心业务模型（用你的例子验证）

```
个人产值 = 日单价 × 在该项目服务天数
项目支出 = Σ(该项目所有员工 日单价 × 服务天数)

例：
A 员工 单价500 × (7/1–7/10 = 10天) = 5000  → 项目1
B 员工 单价400 × (7/5–7/10 = 6天)  = 2400  → 项目2
C 员工 单价200 × (7/8–7/10 = 3天)  = 600   → 项目1
→ 项目1 支出 = 5600；项目2 支出 = 2400
```

> 这个模型是整个系统的"主心骨"，所有功能围绕它展开。

## 三、需求池与优先级建议

按"先跑通主线、再补管控与体验"的原则分级：

| 优先级 | 需求 | 说明 | 复杂度 |
|--------|------|------|--------|
| **P0** | 员工档案 + 日单价管理 | 姓名、岗位、日单价（可生效日期） | 低 |
| **P0** | 项目管理 + 项目预算 | 项目名、预算金额、负责人 | 低 |
| **P0** | 行程/服务记录录入 | 员工 + 项目 + 起止日期（自动算天数） | 中 |
| **P0** | 自动核算引擎 | 日单价×天数=个人产值；聚合=项目支出 | 中 |
| **P0** | 总览看板 | 员工在服务项目 / 个人产值 / 项目预算 / 执行率 | 中 |
| **P1** | 预算管控 | 预算设置、超支预警、预算调整记录 | 中 |
| **P1** | 多项目多人分摊 | 同一员工跨项目、同一项目多人，正确聚合 | 中 |
| **P1** | 报表导出 | 按员工/项目/时间段导出 Excel | 低 |
| **P1** | 权限与角色 | 管理员/员工/财务 数据范围隔离 | 中 |
| **P2** | 移动端填报 | 微信小程序/H5，出差就地录入 | 高 |
| **P2** | 差旅报销模块 | 车票/住宿等实报实销，并入项目支出 | 中 |
| **P2** | 对客户开票回款 | 合同/客户/回款（范围明显变大） | 高 |

> 建议 **MVP = 所有 P0 + 预算管控(P1)**，先把"算得清"跑通，再迭代 P2。

## 四、可行的计划清单（供审核）

> 假设：技术路线采用"若依系二开"（首选 `oa-flow-pms` 或 RuoYi-Vue-Plus）；先 PC 端 MVP，移动端与差旅报销作为后续迭代；部署形态先本地/内网跑通验证。
> 以下清单可直接转为项目事项（见文末提醒）。

### 阶段 0：确认与立项（约 0.5 周）
- [ ] **决策技术底座**：确认用 `oa-flow-pms` 二开 还是 RuoYi-Vue-Plus 自写成本模块
- [ ] **确认费用口径**：仅人工产值 / 人工+差旅 / 含开票（影响数据模型）
- [ ] **确认部署形态**：本地验证 / 内网私有 / 公有云（影响架构）
- [ ] **确认范围边界**：MVP 是否包含预算超支预警、权限分级
- [ ] 组建最小团队：产品(你) + 1 名研发 + （可选）1 名财务对接人

### 阶段 1：MVP 核心 — 数据模型与核算（约 1.5–2 周）
- [ ] 设计数据表：员工(含日单价)、项目(含预算)、服务记录(员工+项目+起止日期)
- [ ] 实现日单价×天数自动核算（含跨项目、跨月、天数计算边界）
- [ ] 实现员工录入/编辑界面（日单价、行程）
- [ ] 实现项目录入/编辑界面（项目、预算）
- [ ] 实现总览看板：员工→在服务项目、个人产值、项目预算 vs 实际支出
- [ ] 用你的 A/B/C 例子做端到端验证

### 阶段 2：预算管控与报表（约 1 周）
- [ ] 项目预算设置与调整留痕
- [ ] 超支预警（看板红字 / 消息提醒）
- [ ] 按员工/项目/时间段导出 Excel
- [ ] 基础权限：管理员可见全部，员工仅见自己

### 阶段 3（迭代）：移动端与出差场景（约 1.5–2 周）
- [ ] 微信小程序/H5 填报：行程+服务天数，出差现场随手录
- [ ] 移动端看板（只读）：当前在服务项目、产值

### 阶段 4（迭代）：差旅报销 / 对客户开票（按需）
- [ ] 差旅报销录入 → 并入项目支出
- [ ] （如需要）客户/合同/回款模块

### 阶段 5：上线与复盘
- [ ] 小范围试点（你 + 2~3 名同事，跑 1~2 个项目）
- [ ] 收集反馈，输出上线复盘报告与后续优化项

## 六、待确认问题清单（审核时请拍板）

以下问题直接影响架构与功能范围，原计划用提问确认，现列出供你一并决定：

1. **费用口径**：仅人工产值（日单价×天数）/ 人工+差旅报销 / 含对客户开票回款？
2. **部署形态**：本地验证 / 公司内网私有部署 / 公有云 SaaS（出差同事访问体验不同）？
3. **移动填报**：MVP 是否需要手机端，还是先 PC 端后续补？
4. **人员规模**：大概多少员工、多少并行项目？（决定是否需要分组织/多公司）
5. **预算口径**：预算是按项目总额，还是按"人工成本预算"单列？超支是否允许？
6. **现有数据**：是否有 Excel/历史台账可导入，避免手工重录？
