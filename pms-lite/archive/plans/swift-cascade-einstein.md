## 1. 技术底座假设（需确认）

- **首选底座**：若依系工程类项目管理系统 `huangxing2010/oa-flow-pms`（基于 RuoYi-Flowable-Plus，Java + Spring Boot + Vue，MIT），自带项目/进度/成本/OA 概念，裁剪最快。
- **备选**：`RuoYi-Vue-Plus` 干净底座自行写成本模型。
- **部署**：首期本地验证（内网/本机），架构预留可迁移公有云。
- **接口**：API 优先设计，预留移动端接口（本版不出移动端）。
- ⚠️ 该技术选型为假设，执行前请确认；若选其他底座，任务拆分不变，仅脚手架任务 0.1 不同。

## 2. 研发任务拆解（Epic → 任务 → 子任务 → 依赖）

### Epic 0：技术底座与环境（前置）
- **T0.1** 拉取基础框架（oa-flow-pms）、初始化本地开发/运行环境（本地验证部署）
- **T0.2** 初始化数据库、若依权限框架、组织/用户体系；确认 API 层可对外暴露（移动端预留）
- 依赖：无

### Epic 1：组织与主数据（P0）
- **T1.1** 部门管理（CRUD + 列表/树）
- **T1.2** 员工管理 + 日单价（含生效日期历史；**单价修改仅管理员**）
- **T1.3** 项目管理（归属部门、负责人、预算）
- 依赖：T0.2 → T1.1（员工/项目归属部门）

### Epic 2：服务记录与核算引擎（P0 + 历史追溯 P1）
- **T2.1** 服务记录录入（员工+项目+起止，自动算天数=截止−起始+1，校验倒序/同项目重叠）
- **T2.2** 核算引擎（个人人工产值 = 服务日期匹配生效单价 × 天数；项目人工 = 聚合）
- **T2.3** 历史单价追溯（按服务日期匹配当时生效单价；无匹配标"单价缺失"置顶）
- 依赖：T1.2、T1.3、T2.1

### Epic 3：项目成本明细（P1）
- **T3.1** 成本明细录入（项目+类型[差旅/材料/外包/其他]+金额+日期+备注，校验≤0/空）
- **T3.2** 成本聚合（项目其他成本 = Σ明细；项目总支出 = 人工 + 其他；重算执行率）
- 依赖：T1.3、T3.1

### Epic 4：总览看板（P0）
- **T4.1** 员工卡片（部门、在服务项目、人工产值）
- **T4.2** 项目卡片（预算/人工/其他/总支出/执行率；超支红字置顶；缓存≤5分钟）
- 依赖：T2.2、T3.2

### Epic 5：预算管控（P1）
- **T5.1** 超支预警（执行率≥80% 黄 / ≥100% 红置顶）
- **T5.2** 预算调整留痕（前后值、操作人、时间；仅管理员）
- 依赖：T1.3、T2.2、T3.2

### Epic 6：权限与角色（P1）
- **T6.1** 四角色数据范围隔离（系统管理员 / 项目负责人 / 部门负责人 / 员工）
- **T6.2** 项目负责人权限：所负责项目成本录入+查看；不可改单价/预算/他人项目
- 依赖：T1.1、T1.3

### Epic 7：导入与导出（P1）
- **T7.1** Excel 导入（员工/项目/服务记录/成本明细 四类模板+逐行校验+失败回执）
- **T7.2** Excel 导出（按员工/项目/时间段）
- 依赖：T1.2、T1.3、T2.1、T3.1

### Epic 8：测试与试点（衔接阶段五）
- **T8.1** 基于 PRD 生成测试用例（含 A/B/C 例子、AC-1~16 修订后）
- **T8.2** 小范围试点（管理员 + 2~3 同事，1~2 个项目验证）
- 依赖：全部 P0/P1

## 4. 关键依赖与风险

- **D1**（硬依赖）：T2.2 核算引擎依赖 T1.2 单价历史 + T2.1 服务记录，必须第2批优先。
- **D2**：T4/T5 依赖 T2.2 + T3.2 聚合结果，看板与预警不能早于核算。
- **R1 技术底座风险**：若依系二开若工作流模块过重，可能需裁剪；建议先验证 oa-flow-pms 是否可直接复用成本/项目模块。
- **R2 单价模型**：已确认"全局单价 + 管理员修改 + 历史追溯"，无需按项目独立定价（降低复杂度）。
- **R3 权限粒度**：四角色 + 部门 + 项目负责人三维隔离，需在 T6.1 设计统一数据权限切面。
